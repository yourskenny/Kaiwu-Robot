# Kaiwu-Robot: Robot Vacuum PPO Agent

基于腾讯开悟（KaiwuDRL）分布式强化学习框架开发的“清扫大作战”智能体。本项目在原有的 `ppo-syg-1-9h` 模型基础上进行了深度重构与优化，旨在提升扫地机器人在复杂环境下的避障、寻污与电量管理能力。

## 🚀 核心优化与当前进度

本项目主要完成了三大核心模块的深度优化，并已将代码调整至支持 Docker 容器内挂载调试的最佳状态。

### 1. 特征工程 (Feature Engineering) 升级
*   **8方向雷达探测**：将全局特征中的 4 向探测升级为 8 向（东南西北 + 四个对角线），并且**同时探测污渍和障碍物**。这使得全局状态维度从 12D 提升到了 24D，大幅提升了智能体避障和寻找污渍的感知能力。
*   **极坐标实体感知**：将充电桩和 NPC（官方机器人）的相对位置由直角坐标（dx, dz）改为了**极坐标（距离，角度）**。这让神经网络更容易学习“转向并靠近”的导航逻辑，而非复杂的平面解析几何。

### 2. 奖励机制 (Reward Shaping) 重构
*   **探索奖励 (Exploration Reward)**：当机器人踏入之前从未访问过的格子时，给予 `+0.01` 的微小正向奖励，鼓励其主动覆盖全图，减少在已清理区域的原地打转。
*   **连击加成 (Combo Reward)**：引入连续清扫机制。当连续多步清扫到污渍时，每多清扫一步额外增加 `0.1` 的连击奖励，引导模型规划出“长条形”的高效清扫路线。
*   **防反复横跳惩罚 (Anti-oscillation)**：如果上一步和当前步方向完全相反（例如向左走完立刻向右走）且未清扫到污渍，将给予 `-0.01` 的惩罚，有效减少无意义的电量消耗和原地摇摆。

### 3. 神经网络结构 (Model Architecture) 演进
*   **多模态融合双流架构**：彻底抛弃了单纯将所有特征展平输入 MLP 的做法。
    *   **CNN 流**：7x7 的局部视野（49D）会被重塑为 `1x7x7` 的图像张量，送入**两层小型 CNN** 提取空间特征（如死角、长廊）。
    *   **MLP 流**：全局和历史状态向量（54D）送入 MLP 提取状态特征。
    *   最后将两股特征进行融合（Concat）。
*   **解耦与稳定化**：将 Actor（策略）和 Critic（价值评估）的网络头**完全解耦**，不再共享特征提取层的参数，防止梯度互相干扰。
*   **LayerNorm & GELU**：将激活函数由 `ReLU` 替换为更加平滑的 `GELU`，并在隐藏层融合后引入了 `LayerNorm`，极大提高了 PPO 在高维输入下的训练稳定性。
*   **维度匹配修复**：修复了因引入 8 向雷达导致的 `103D` 特征张量与配置不匹配的问题，当前观测维度固定为 103。

## 🛠️ 如何运行与开发

本赛题依赖于腾讯开悟官方提供的 `kaiwudrl` 分布式强化学习框架，因此**所有的验证和训练都必须在 Docker 容器内进行**。

### 1. 启动容器编排
进入 `dev` 目录，通过 docker-compose 拉起环境核心（gamecore）和训练容器（kaiwudrl）：
```powershell
cd dev
docker compose --env-file .env -f .docker-compose.yaml up -d
```
*(注：`.env` 文件中已配置 `KAIWU_CODE_FILE=../code`，容器启动时会将宿主机的 `code` 目录挂载到容器的 `/workspace/code`。因此日常开发请在外部直接修改，容器内会实时生效。)*

### 2. 验证模型正确性
容器启动后，在宿主机执行以下命令，让容器内的环境运行一次前向和反向传播测试，验证维度与逻辑是否报错：
```powershell
docker compose exec kaiwudrl python3 /workspace/code/train_test.py
```
如果输出 `run train_test success`，则说明当前代码已完美跑通！

### 3. 提交与打包
当需要提交模型时，通过客户端或相关评测脚本调用 `save_model`，框架会将模型检查点（必须包含 `model.ckpt-{id}.pkl`）与智能体代码包一并打包为 ZIP 供提交。
